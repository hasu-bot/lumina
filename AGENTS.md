# Repository Guidance

Before making product, UX, business, or routing decisions for Creative YOLO,
read the ecosystem source documents first.

Primary references:
- `yolo-members/docs/creative-yolo/00-MASTER-PLAN.md` for mission, roles, monetization, roadmap, and decision criteria.
- `yolo-members/docs/creative-yolo/01-ECOSYSTEM-IA.md` for site structure, cross-links, domains, and data ownership.
- `yolo-members/docs/creative-yolo/02-YOLO-OS.md` for operations and SOPs.
- `yolo-members/docs/creative-yolo/03-AI-AGENTS.md` for AI agent boundaries and rollout order.
- `yolo-members/docs/creative-yolo/04-UX-AUDIT.md` for prioritized UX issues.
- `yolo-members/docs/creative-yolo/05-LONGTERM-ROADMAP.md` for film festival, lumina, and creator-agency planning.

For lumina-specific work, also read:
- `docs/LUMINA-BUSINESS.md`
- `docs/LUMINA-IP.md`

Use these documents as the source of truth. If implementation and docs differ,
surface the mismatch before making a broad change.
