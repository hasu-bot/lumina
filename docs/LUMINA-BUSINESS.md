## 1. このシステムの戦略的な意味

lumina のマッチングMVP（モデル一覧→30分枠予約→ダッシュボード）は、単なる写真展の付属機能ではなく、
**将来のモデル/クリエイター事務所事業のプロトタイプ**である。

```
2026  会期限定マッチング（現MVP）          ← いまここ
2027  creator_type 拡張（カメラマン・デザイナーも参加者に）
2028  projects テーブル実装 → 通年マッチング（撮影会/MV/キャスト募集）
2029  事務所機能（登録制→専属契約）
```

READMEの「将来拡張（未実装・設計のみ）」に書かれた `creator_type` / `projects` は、この順で実装する。

## 2. 写真展の事業設計（年次）

### 収支モデル（目安）
| 項目 | 設計 |
|---|---|
| 会場費 | ギャラリー1〜2週間：5〜15万円 |
| 収入1：出展料 | 1枠（壁面＋名鑑掲載）3,000〜5,000円 × 20〜30枠 |
| 収入2：撮影枠 | 30分枠の料金設定機能は実装済み。モデルへの分配を主、運営フィー10〜20% |
| 収入3：物販 | プリント・ZINE 委託販売（手数料20%） |
| 目標 | 単年トントン。目的は出会いとデータ（マスタープラン §4 の原則どおり） |

### 検証すべきデータ（2026会期で必ず取る）
- 予約枠 vs 当日枠の消化率（50%交互配置ルールの妥当性検証）
- 1予約あたりの平均成立時間帯・人気ジャンル
- 来場者→Discord 参加への転換数（会場に導線を置いた場合）
- **このデータが2027年の年2回化・2028年の通年化の判断材料**

## 3. 表示・信頼ルール（事業の生命線）

READMEの表示ルールを事業原則に昇格させる：

1. **所属事務所を必ず明記**（カード・詳細とも）。無所属は「フリー」
2. **「YOLOメンバー」表記はしない**——YOLOは主催ブランド表記のみ。他事務所のタレントをYOLOの実績のように見せない
3. 予約者情報は名前のみ最小収集（現設計を維持）。連絡先を集め始めるときは利用目的の明示とセット
4. 未成年モデルの参加は保護者同意を必須化（2027年の登録制までに規約整備）

この「他者の看板を尊重する」姿勢が、2029年に専属契約を始めたとき、地元事務所と共存できるかどうかを決める。

## 4. `projects`（募集案件）拡張の設計方針（2028）

```
projects
  id / type（撮影会 | MV | 映画キャスト | その他）
  title / description / date_range / location
  reward（謝礼の有無・金額目安）※必須：無償案件は「無償」と明記させる
  owner（募集主：メンバー or 相談サイト経由の外部依頼）
  status（募集中 | 選考中 | 成立 | 終了）

applications
  project_id / model_id（将来 creator_id）/ message / status
```

- 入口は2つ：メンバーの自主企画（Discord発）と、**相談サイト経由の外部案件**（yolo-soudan との接続点）
- 応募者側は yolo-members のプロフィール（依頼OK/コラボOKフラグ）と突合できるようにする
- **謝礼の透明化を必須にする**（「相談して決める」は可、空欄は不可）。無償搾取マッチングの温床にしないこと
